3. Writing JSON

Writing JSON requires following its strict syntax to ensure itâ€™s valid for data exchange.

  • Rules:
    • Keys must have double quotes ("key").
    • Values can be strings (double-quoted), numbers, arrays, objects, booleans, or null.
    • No functions or methods allowed.
  • Tip: Practice writing complex JSON with nested objects and arrays.

Example: Complex JSON


{
    "user": {
        "name": "Alice",
        "age": 30
    },
    "posts": [
        { "id": 1, "title": "Hello World" },
        { "id": 2, "title": "JSON Rules" }
    ],
    "active": true
}
