6. Debugging with Network Dev Tool

Use Chrome Network Dev Tool to debug failed requests, inspect response headers, and measure request timing.

  • Debugging Tips: Look for red status codes (e.g., 404), check “Response” tab for errors, use “Timing” to analyze delays.
  • Common Issues: Wrong URL, server errors, CORS issues.

Example: Debugging a Bad Request

Try a bad URL:


fetch("https://swapi.co/api/invalid")
    .then(res => res.json())
    .catch(err => console.log("Error:", err));

Check Network tab for a 404 error.

Mini Exercise (5 min): Run the code above, open Network Dev Tool, and find the failed request. Note the status code (404) and error message in the “Response” tab.
