5. Introduction to Chrome Network Dev Tool

The Chrome Network Dev Tool lets you inspect HTTP requests, responses, and errors in Chromeâ€™s Developer Tools (Inspect > Network tab).

  • Why Use It? Debug API calls, check status codes (e.g., 200 OK, 404 Not Found), and view JSON responses.
  • Key Features: Filter requests, view headers, preview responses.

Example: Inspecting a Fetch Request

Inspect a SWAPI request:


fetch("https://swapi.co/api/people/1")
    .then(res => res.json())
    .then(data => console.log(data.name));

Open Dev Tools (F12 on Windows) (command + i, on a Mac), go to Network, run the code, and check the request details.

Mini Exercise (5 min): Run the fetch code above in a browser console. Open Chrome Network Dev Tool, find the request to people/1, and note the status code (should be 200) and response JSON.
