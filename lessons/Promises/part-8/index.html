Promises: Synchronous vs. Asynchronous Programming

8. Async/Await and Conclusion

Async/Await (ES2017) simplifies promise-based code, making asynchronous operations look synchronous using the async and await keywords.

  • Async: Marks a function that returns a promise.
  • Await: Pauses execution inside an async function until a promise resolves.
  • Conclusion: Callbacks, promises, and async/await enable non-blocking code, with async/await being the most readable.

Example: Async/Await with SWAPI

Fetch Luke’s homeworld name:


async function getData() {
    const lukeResponse = await fetch("https://swapi.co/api/people/1");
    const lukeData = await lukeResponse.json();
    const planetResponse = await fetch(lukeData.homeworld);
    const planetData = await planetResponse.json();
    console.log(planetData.name); // "Tatooine"
    return lukeData;
}
getData().then(data => console.log(data.name));
