Promises: Synchronous vs. Asynchronous Programming

7. Promise Chaining

Promise chaining allows sequential asynchronous operations without nesting, as .then() returns a new promise, passing its return value to the next .then().

  • Benefit: Avoids callback hell, improves readability.
  • Mechanism: Return values (or promises) from one .then() feed into the next.

Example: Promise Chaining with SWAPI

Fetch Luke’s homeworld name:


function getData(url) {
    return fetch(url).then(res => res.json());
}
getData("https://swapi.co/api/people/1")
    .then(lukeData => {
        console.log(lukeData.homeworld);
        return getData(lukeData.homeworld);
    })
    .then(planetData => console.log(planetData.name)); // "Tatooine"
