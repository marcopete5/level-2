Promises: Synchronous vs. Asynchronous Programming

6. Promises and States

Promises are objects representing future values, avoiding callback nesting. They have three states: Pending, Resolved (fulfilled, handled by .then()), and Rejected (failed, handled by .catch()).

  • Pending: Initial state, waiting for resolution or rejection.
  • Resolved: Operation succeeded, returns data.
  • Rejected: Operation failed, returns an error.

Example: Promise States

Handle a fetch promise:


fetch("https://swapi.co/api/people/1")
    .then(res => res.json())
    .then(data => console.log(data.name)) // Resolved
    .catch(err => console.log("Error:", err)); // Rejected
