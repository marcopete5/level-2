Promises: Synchronous vs. Asynchronous Programming

5. Callbacks

Callbacks are functions passed as parameters to execute later, enabling asynchronous behavior but risking “callback hell” with nesting.

  • Examples: setTimeout, addEventListener, custom API calls.
  • Issue: Deep nesting can make code hard to read (see callbackhell.com).

Example: Callback with SWAPI

Fetch Luke Skywalker’s homeworld name:


function getData(url, callback) {
    const xhr = new XMLHttpRequest();
    xhr.onreadystatechange = function() {
        if (xhr.readyState === 4 && xhr.status === 200) {
            callback(JSON.parse(xhr.responseText));
        }
    };
    xhr.open("GET", url, true);
    xhr.send();
}
getData("https://swapi.co/api/people/1", lukeData => {
    getData(lukeData.homeworld, planetData => {
        console.log(planetData.name); // "Tatooine"
    });
});
