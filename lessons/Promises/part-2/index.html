Promises: Synchronous vs. Asynchronous Programming

2. Synchronous Code Examples

Synchronous code, or blocking code, is like waiting in a single line at a store. Each task must finish before the next one starts, making the order predictable but possibly slow if a task takes a long time.

  • Characteristics: Each line of code waits for the one before it to complete.
  • Examples: Setting variables, running simple functions, or doing math calculations.

Example: Synchronous Code

Each line runs only after the previous one is done:


var firstName = "Joe"; // Sets a name
function addExcitement(str) {
    return str + "!!1!";
}
console.log(addExcitement("Hi")); // Logs "Hi!!1!"
function factorial(n) {
    if (n === 1) return 1;
    return n * factorial(n - 1);
}
console.log(factorial(5)); // Logs 120

Breaking Down the Factorial Function (Super Simple!)

The factorial function calculates something called a "factorial." A factorial is when you multiply a number by every number below it down to 1. For example, the factorial of 5 (written as 5!) is 5 × 4 × 3 × 2 × 1 = 120. Let’s see how the factorial function does this in a way that’s easy to follow, like stacking blocks!

  • What the Code Does: The function factorial(n) takes a number (like 5) and gives back its factorial (like 120).
  • Two Parts to the Function:
    • Stopping Point (Base Case): If the number is 1, it just returns 1. This is like saying, “If we’re at the smallest block, stop here!”
    • Building the Answer (Recursion): For any other number, it multiplies the number by the factorial of the number just below it. This is like saying, “Take this block and stack it on top of a smaller pile.”
  • Let’s Pretend We Run factorial(3):

    Imagine you’re building a tower of blocks, and each block needs a smaller tower below it. Here’s what happens step-by-step:

    1. Start with 3: The function sees n = 3. It’s not 1, so it says, “I need to do 3 * factorial(2).” It waits to figure out what factorial(2) is.
    2. Move to 2: Now it runs factorial(2). Since 2 isn’t 1, it says, “I need 2 * factorial(1).” It waits for factorial(1).
    3. Reach 1: For factorial(1), the function sees n = 1 and returns 1. This is the smallest block!
    4. Build Back Up:
      • Back to factorial(2): It now knows factorial(1) = 1, so it does 2 * 1 = 2.
      • Back to factorial(3): It now knows factorial(2) = 2, so it does 3 * 2 = 6.
    5. Final Answer: factorial(3) returns 6, which is 3! (3 × 2 × 1).
  • Why Is This Synchronous? The function is like a patient worker who won’t move on until each smaller tower is built. It waits for each factorial call to finish before doing the multiplication, making it “blocking” because no other code can run until the whole tower (answer) is complete.
  • What About factorial(5)? It works the same way but with more steps: 5 * factorial(4) waits for 4 * factorial(3), which waits for 3 * factorial(2), then 2 * factorial(1), and finally 1. Then it multiplies back up: 1, 2, 6, 24, 120. The result is 120!

Think of the factorial function as a game of passing a question down (“What’s the factorial of the next number?”) and then building the answer back up, one step at a time, without skipping ahead.
