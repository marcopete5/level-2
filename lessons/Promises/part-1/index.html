Promises: Synchronous vs. Asynchronous Programming

This module explores synchronous and asynchronous code, key concepts for building responsive web applications. You’ll learn how to write non-blocking code using callbacks, promises, and async/await.

1. Synchronous vs. Asynchronous

Synchronous code runs sequentially, blocking other code until complete, like waiting at a single cashier. Asynchronous code starts tasks without blocking, like using self-checkout kiosks, allowing other code to run.

  • Synchronous (Blocking): Each operation waits for the previous one to finish.
  • Asynchronous (Non-Blocking): Operations start but don’t halt other code, enabling faster, responsive apps.

Example: Sync vs. Async

Synchronous code waits:


console.log("Start");
console.log("Processing...");
console.log("End");

Asynchronous code doesn’t:


console.log("Start");
setTimeout(() => console.log("Processing..."), 500);
console.log("End");

Mini Exercise: Run the async example above in your browser’s console. Note the order of the logs (e.g., “Start”, “End”, “Processing...”). Why does “Processing...” appear last?
