Using Axios for HTTP Requests

Understanding Axios

Axios is like a helpful messenger that helps us talk to the internet. It makes it easy for our code to ask for information from a website or send information to it.

Getting Todos from the Website

Imagine we have a list of things to do called "todos" on a website. We can use Axios to ask the website to give us all the todos it has. 


axios.get("<https://api.vschool.io/scrimbalessons/todo>")
    .then(response => {
        for(let i = 0; i < response.data.length; i++){
            const h1 = document.createElement('h1')
            h1.textContent = response.data[i].title
            document.body.appendChild(h1)
        }
    })
    .catch(error => console.log(error))

In this code:

  • axios.get() asks the website to give us all the todos.
  • When the website responds, we go through each todo and add its title to our webpage.
  • If something goes wrong, like the internet connection fails, we'll see an error message.

Getting One Specific Todo

Sometimes we only want one specific todo. We can tell the website the ID of the todo we want, and it will give us just that one. 


axios.get("<https://api.vschool.io/scrimbalessons/todo/5d8bd511ee91575e6d49e06e>")
     .then(response => console.log(response.data))
     .catch(error => console.log(error))

In this code:

  • axios.get() asks the website to give us one todo with a specific ID.
  • When the website responds, we show the information about that todo.
  • If something goes wrong, we'll see an error message.

Conclusion

Axios is our friendly helper for talking to websites. With it, we can easily ask for information from the internet and handle the responses. This example showed how to use Axios to get a list of todos from a website and also how to get just one specific todo.
