Postman Assignment: Exploring APIs

In this assignment, you’ll use Postman to practice HTTP requests with the Star Wars API (SWAPI) and the V School API (https://api.vschool.io/[your_name]/todo). Complete the tasks below to master GET, POST, PUT, and DELETE requests.

Assignment: Postman Practice

Follow these tasks to explore APIs and test your skills with Postman. Use your own name in the V School API URL (e.g., fred for https://api.vschool.io/fred/todo).

Tasks

  • Task 1: GET Planets (SWAPI)
    • Use Postman to send a GET request to https://swapi.dev/api/planets/ to retrieve a list of planets.
    • Note one planet’s name from the response.
  • Task 2: GET Person (SWAPI)
    • Send a GET request to https://swapi.dev/api/people/1/ to retrieve a single person object.
    • Note the person’s name from the response.
  • Task 3: POST Todos (V School API)
    • Send three POST requests to https://api.vschool.io/[your_name]/todo to create three new todo objects. Use this JSON template: 
      • 
{
    "title": "This is required",
    "description": "This is optional",
    "price": 0,
    "imgUrl": "http://path-to-url.jpg",
    "completed": false
}
    • Set Body to raw and JSON (application/json). Vary the title for each todo.
    • Send a GET request to the same URL to confirm your todos are listed.
  • Task 4: GET Single Todo (V School API)
    • From the GET response in Task 3, find a todo’s _id.
    • Send a GET request to https://api.vschool.io/[your_name]/todo/[_id] to retrieve that todo.
    • Confirm the todo’s title.
  • Task 5: DELETE Todo (V School API)
    • Using the same _id from Task 4, send a DELETE request to https://api.vschool.io/[your_name]/todo/[_id].
    • Verify the response includes {"msg": "Successfully deleted record"}.
  • Task 6: PUT Todo (V School API)
    • From the GET in Task 3, select another todo’s _id.
    • Send a PUT request to https://api.vschool.io/[your_name]/todo/[_id] with a JSON body updating at least one field (e.g., {"completed": true}).
    • Set Body to raw and JSON (application/json). Ensure valid JSON.
    • Confirm the updated todo in the response.

Tips

  • Validate JSON: Use JSONLint to check your POST and PUT JSON bodies.
  • Check Responses: If a request fails, inspect the response for error messages (e.g., invalid JSON).
  • Save Requests: Save your Postman requests for easy reuse.
